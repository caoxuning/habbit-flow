# HabitFlow 2026-06-23 功能增删说明

> 日期：2026-06-23  
> 说明范围：本次说明汇总当天围绕任务日历、打卡分享、好友监督、分层智能提醒、首页统计与成长日志联动所做的功能扩展和问题修复。

## 一、任务日历功能优化

### 1. 日历目标摘要展示优化

原任务日历会在同一天日期格中直接展示当天所有目标。当用户当天目标数量较多时，日期格内容容易拥挤，影响页面显示效果。

本次优化后，任务日历每天最多只展示两类摘要目标：

- 紧急目标显示一个。
- 其他类型目标显示一个。

如果当天实际目标数量超过日历格可展示数量，系统会在日期格中显示省略号和当天目标总数，例如：

```text
...
共 5 项目标
```

通过这种方式，用户可以快速感知当天还有更多任务，同时避免日历页面内容过度堆叠。

### 2. 日期点击查看完整日程

任务日历中的每一天现在都可以点击。用户点击某个日期后，页面会弹出日程详情窗口。

弹窗中展示当天完整任务安排，包括：

- 当天需要完成的全部目标。
- 目标名称。
- 目标类型。
- 目标周期。
- 每日目标次数。
- 目标优先级。
- 当天是否已经打卡。
- 若已打卡，展示打卡备注。

该功能解决了“日历格只展示摘要，但用户仍需要查看完整任务”的问题。

## 二、打卡分享机制扩展

### 1. 打卡成功后增加分享入口

用户完成今日打卡后，系统会先展示原有打卡成功激励卡片。用户关闭激励卡片后，系统会弹出“分享本次打卡”窗口。

用户可以选择：

- 不分享。
- 分享到圈子动态。
- 通知好友。
- 仅通知好友但不发布圈子动态。

该功能将个人打卡行为和社交互动连接起来，使打卡不再只是个人记录，也可以转化为好友监督和圈子动态。

### 2. 支持动态可见范围

打卡分享时新增动态可见范围设置，包含：

- 公开。
- 好友可见。
- 圈子可见。
- 仅自己可见。

如果用户选择“仅自己可见”，系统不会发布圈子动态，但仍允许用户保留好友通知，从而实现“只提醒好友，不公开发布”的场景。

### 3. 打卡动态与打卡记录关联

圈子动态表新增关联打卡编号字段。用户通过打卡分享生成的动态会记录对应打卡记录编号，方便后续区分普通动态和打卡动态。

动态类型新增：

- 普通动态。
- 打卡分享动态。

## 三、好友监督机制扩展

### 1. 好友打卡消息通知

用户完成打卡并选择通知好友后，系统会自动向指定好友发送私信消息。

消息会说明：

- 哪个用户完成了打卡。
- 完成的是哪个目标。
- 好友可以进入动态或聊天中进行鼓励。

该功能增强了好友之间的监督感和反馈感。

### 2. 好友今日打卡看板

社交页面新增“好友今日打卡看板”。

看板展示每位好友当天的打卡状态，包括：

- 好友是否已打卡。
- 今日打卡次数。
- 最近完成的目标。
- 当前进行中目标数量。

对于已打卡好友，用户可以发送鼓励消息。  
对于未打卡好友，用户可以发送提醒消息。

该功能让好友监督从聊天功能扩展为可视化状态看板。

### 3. 动态点赞、评论与鼓励

圈子动态和打卡动态继续支持点赞、评论功能，并新增“鼓励”按钮。

用户点击鼓励后，系统会自动生成一条鼓励评论，例如：

- 坚持得很稳。
- 今天也完成得很漂亮。
- 明天继续一起打卡。

评论和点赞会向动态发布者产生互动通知，形成“打卡 -> 分享 -> 互动 -> 监督”的闭环。

## 四、分层智能提醒机制扩展

### 1. 新增消息提醒模块

系统新增消息提醒表，用于存储系统主动生成的提醒消息。

提醒消息包含：

- 提醒编号。
- 用户编号。
- 提醒类型。
- 提醒标题。
- 提醒内容。
- 是否已读。
- 创建时间。

前端新增“消息提醒”页面，侧边栏菜单显示未读数量角标。

### 2. 日常打卡提醒

系统会检测用户当天应完成但尚未打卡的目标。

如果当天存在未完成目标，则生成日常打卡提醒，提示用户及时完成今日计划。

示例：

```text
你今天还有 2 个目标未打卡：每日背单词、晚间拉伸。记得按计划完成。
```

### 3. 目标到期预警

系统会检测三天内即将到期的进行中目标。

如果目标即将结束，则生成目标到期预警，提醒用户及时完成剩余任务。

示例：

```text
目标「阅读专业书 20 页」还剩 2 天，请及时完成剩余打卡或调整计划。
```

### 4. 连续打卡中断提醒

系统会检测用户昨天应打卡但未完成的目标。

如果存在中断情况，则生成连续打卡中断提醒，提示用户今天重新开始，避免持续中断。

示例：

```text
你昨天有 1 个计划未打卡：跑步 3 公里。今天可以重新开始，避免持续中断。
```

### 5. 提醒消息管理

消息提醒页面支持：

- 查看全部提醒。
- 只看未读提醒。
- 重新生成提醒。
- 单条标记已读。
- 全部标记已读。
- 删除提醒。

系统会按当天、用户、提醒类型和标题进行去重，避免重复刷新时产生大量相同提醒。

## 五、首页统计图与成长日志问题修复

### 1. 修复成长日志初次不显示问题

问题表现：

用户打开网页后，首页两个统计图可以显示，但成长日志预览没有内容。进入成长日志页面后，需要手动修改查询时间，成长日志才会显示。

问题原因：

首页加载时已经请求了成长日志数据，但前端没有把接口返回的成长日志数据赋值给 `timelineList`，导致页面初始状态为空。

修复方式：

在首页统一加载数据时，将成长日志接口返回的 `timelineData` 正确写入 `timelineList`。

### 2. 修复返回首页后统计图不显示问题

问题表现：

用户进入成长日志页面并查询后，再回到首页，原本显示正常的两个统计图无法显示。

问题原因：

首页图表使用 ECharts 渲染，页面通过 `v-if` 切换后，原图表 DOM 会被销毁。再次回到首页时，前端仍复用旧的 ECharts 实例，导致实例绑定的 DOM 已经失效，从而无法正常渲染。

修复方式：

回到首页时重新检查当前图表 DOM，如果旧实例绑定的 DOM 已失效，则销毁旧实例并基于新 DOM 重新初始化 ECharts 图表。

## 六、数据库结构变更

### 1. 圈子动态表扩展

圈子动态表新增字段：

- 关联打卡编号。
- 动态可见范围。
- 动态类型。

用于支持打卡分享动态、好友可见动态、圈子可见动态和私密动态。

### 2. 新增消息提醒表

新增消息提醒表 `notification`，用于支撑日常打卡提醒、目标到期预警和连续打卡中断提醒。

系统启动时会自动创建新表。若旧数据库中已经存在消息提醒表但缺少新字段，系统会自动补充提醒类型和提醒标题字段。

## 七、接口扩展

### 1. 打卡分享相关接口

新增：

```http
POST /api/social/checkins/share
```

用于将打卡记录分享为圈子动态，并可同步通知好友。

### 2. 好友今日打卡看板接口

新增：

```http
GET /api/social/friends/checkins/today
```

用于查询好友当天打卡状态。

### 3. 消息提醒接口

新增：

```http
GET /api/notifications
GET /api/notifications/unread-count
POST /api/notifications/generate
PUT /api/notifications/{notificationId}/read
PUT /api/notifications/read-all
DELETE /api/notifications/{notificationId}
```

用于完成提醒生成、列表查询、未读统计、已读处理和删除操作。
