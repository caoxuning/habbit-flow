# HabitFlow 项目功能清单

> 版本：v2.0
> 说明：涵盖前端 (Vue3 + Element Plus) 和后端 (FastAPI) 所有功能

---

## 一、用户系统

### 1.1 注册 ✅
- 输入用户名、密码、邮箱
- 后端密码加密存储 (SHA-256)
- 用户名唯一性校验
- 注册成功后自动登录

### 1.2 登录 ✅
- 输入用户名、密码
- 后端返回 JWT Token
- Token 存储在 `localStorage`
- 自动跳转到数据概览页面

### 1.3 退出登录 ✅
- 清除 `localStorage` 中的 Token 和用户信息
- 跳转回登录页

### 1.4 个人信息维护 ✅
- 查看当前用户名、邮箱
- 修改用户名和邮箱
- 更新 `localStorage` 中的用户信息

### 1.5 修改密码 ✅
- 输入原密码 + 新密码
- 后端验证原密码正确后更新

## 二、目标管理

### 2.1 目标列表 ✅
- 以表格形式展示所有目标
- 显示：名称、类型、周期、日期、完成率、状态
- 完成率以进度条展示

### 2.2 创建目标 ✅
- 弹出对话框填写：
  - 目标名称（如：每天运动30分钟）
  - 目标类型（如：运动、学习、阅读）
  - 开始/结束日期
  - 目标周期（每日/每周/每月）
  - 每日目标次数
  - 状态（进行中/已暂停/已完成）

### 2.3 编辑目标 ✅
- 在弹出的对话框中预填已有数据
- 更新后刷新列表

### 2.4 删除目标 ✅
- 弹出确认框
- 确认后删除目标及其打卡记录（级联）

### 2.5 快速打卡 ✅
- 在目标列表行直接点击「打卡」按钮
- 自动记录打卡，备注为"快速打卡"
- 打卡成功后弹出精选内容弹窗

---

## 三、打卡记录

### 3.1 今日打卡 ✅
- 选择目标 + 填写备注
- 只能打卡当天日期
- 同一目标同一天不能重复打卡

### 3.2 补卡 ✅
- 选择目标 + 补卡日期 + 原因
- 补卡日期必须早于今天
- 补卡记录会被标记为 `makeup=true`

### 3.3 打卡记录列表 ✅
- 展示所有打卡记录（日期倒序）
- 显示：日期、目标、备注、类型（打卡/补卡）
- 补卡用 warning 标签，正常用 success 标签

### 3.4 数据导出 ✅
- 仅在「打卡记录」页面显示「导出打卡记录」按钮
- 导出 Excel (.xlsx) 格式
- 包含字段：日期、目标名称、目标类型、备注、打卡类型、打卡时间
- 自动下载到本地
- 导出时跳过 axios JSON 拦截器

---

## 四、数据概览（仪表盘）

### 4.1 统计指标卡片 ✅
- 目标总数
- 进行中目标数
- 总完成次数
- 连续打卡天数
- 平均完成率（%）

### 4.2 月度成长报表图表 ✅
- ECharts 柱状图
- 展示近 6 个月的打卡完成次数

### 4.3 目标完成率图表 ✅
- ECharts 折线图 + 面积图
- 展示每个目标的完成率（0-100%）

### 4.4 成长日志预览 ✅
- 在图表下方展示前 7 条打卡记录
- 使用 Element Plus Timeline 组件
- 显示：日期、目标、备注
- 点击「查看全部」跳转到完整成长日志页面

---

## 五、成长日志时间线

### 5.1 时间线视图 ✅
- 以日期倒序排列
- 使用 `<el-timeline>` 组件

### 5.2 事件类型 ✅
- **打卡事件**：显示目标名称、备注、补卡标签
- **勋章事件**：显示勋章名称和描述（绿色标签）
- **每日推荐**：显示当天的精选内容（带背景色）

### 5.3 时间范围筛选 ✅
- 支持：7天 / 30天 / 90天 / 全部（365天）
- 切换后自动重新加载数据

### 5.4 自动加载 ✅
- 切换到「成长日志」页面时自动加载数据
- 主页刷新时同时加载 timeline 数据

## 六、内容推荐系统

### 6.1 打卡成功弹窗 ✅
- 使用 `ElMessageBox.alert()` 弹窗
- 自定义 HTML 内容

### 6.2 推荐内容来源 ✅
- 名人名言（原文 + 中文翻译）
- 英语词汇（单词 + 中文解释 + 例句）
- 学习技巧（技巧说明）

### 6.3 同路人寄语 ✅
- 展示其他用户在同一类型目标的打卡备注
- 最多展示 3 条
- 显示内容 + 目标来源

### 6.4 内容分类匹配 ✅
- 根据目标类型匹配合适的内容
- 如：运动类型 → 运动相关名言/健身技巧
- 如：英语类型 → 英语词汇/学习技巧

## 七、勋章奖励系统

### 7.1 勋章列表 ✅
- 卡片式展示所有已获得勋章
- 显示：图标、名称、描述、获取条件

### 7.2 自动发放 ✅
- 打卡成功后自动检测发放条件
- 发放条件：
  - 首次打卡 → 初次打卡勋章
  - 连续 7 天 → 连续7天勋章
  - 连续 30 天 → 连续30天勋章
  - 累计 100 次 → 百次打卡勋章
  - 累计 100次+连续30天 → 自律达人勋章

### 7.3 空状态提示 ✅
- 未获得任何勋章时显示提示文案

## 九、后端 API 清单

| 路由 | 方法 | 功能 |
|:---|:---:|:---|
| `/api/auth/login` | POST | 登录 |

| `/api/goals/{id}` | DELETE | 删除目标 |
| `/api/check-ins` | GET | 打卡记录列表 |
| `/api/check-ins` | POST | 提交打卡 |
| `/api/check-ins/makeup` | POST | 补卡 |
| `/api/stats/dashboard` | GET | 仪表盘数据 |
| `/api/stats/timeline` | GET | 成长日志时间线 |
| `/api/badges/mine` | GET | 我的勋章 |
| `/api/exports/checkins` | GET | 导出打卡记录(Excel) |
| `/api/social/users/search` | GET | 搜索用户 |
| `/api/social/friends` | GET | 好友列表 |
| `/api/social/friend-requests` | GET | 好友请求列表 |
| `/api/social/friend-requests` | POST | 发送好友请求 |
| `/api/social/friend-requests/{id}/accept` | PUT | 接受好友请求 |
| `/api/social/friend-requests/{id}/reject` | PUT | 拒绝好友请求 |
| `/api/social/circles` | GET | 圈子列表 |
| `/api/social/circles` | POST | 创建圈子 |
| `/api/social/circles/{id}/join` | POST | 加入圈子 |
| `/api/social/circles/{id}/leave` | DELETE | 退出圈子 |
| `/api/social/circles/{id}/posts` | GET | 圈子帖子 |

## 十、前端页面路由

| 视图 | 路由名 | 功能 |
|:---|:---|:---|
| 登录/注册 | `auth` | 未登录时显示 |
| 数据概览 | `dashboard` | 仪表盘 + 图表 + 时间线预览 |
| 目标管理 | `goals` | 目标 CRUD + 快速打卡 |
| 打卡记录 | `checkins` | 打卡/补卡 + 记录列表 + 导出 |
| 成长日志 | `timeline` | 时间线 + 事件类型展示 + 天数筛选 |
| 勋章奖励 | `badges` | 已获得勋章卡片 |
| 个人中心 | `profile` | 个人信息 + 修改密码 |
