# 数据库协作与中期演示方案

## 当前能做到什么

当前后端使用 FastAPI + SQLAlchemy + MySQL。所有业务数据都落在同一个 MySQL 数据库中，包括用户、目标、打卡、勋章、好友、圈子、帖子、点赞、评论。

现在可以做到：

- 多个前端同时访问同一个后端服务。
- 多个后端服务连接同一个 MySQL 数据库。
- 注册、登录、目标管理、每日打卡、补卡、统计看板、勋章、好友、圈子、帖子、点赞、评论。
- 通过脚本生成稳定的中期展示数据。

还没有做到：

- 线上云数据库部署和账号权限隔离。
- 管理员后台。
- 数据库迁移工具，例如 Alembic。
- 多环境配置管理，例如 dev、test、prod 独立配置。

## 大家一起访问数据库怎么实现

推荐中期展示采用“统一演示数据库”方案：

1. 找一台大家都能访问的 MySQL。
   - 可以是组长电脑的 MySQL，但需要同一局域网。
   - 更推荐云服务器或云数据库，展示时更稳定。
2. 在 MySQL 中创建数据库和专用账号。
3. 三个人本地启动后端时，把 `DATABASE_URL` 指向同一个数据库。
4. 前端统一访问自己的本地后端，或者统一访问一台公共后端。

示例连接串：

```text
mysql+pymysql://habitflow_demo:你的密码@数据库IP:3306/habitflow?charset=utf8mb4
```

Windows PowerShell 临时设置：

```powershell
$env:DATABASE_URL="mysql+pymysql://habitflow_demo:你的密码@数据库IP:3306/habitflow?charset=utf8mb4"
```

MySQL 建库和授权示例：

```sql
CREATE DATABASE IF NOT EXISTS habitflow CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'habitflow_demo'@'%' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON habitflow.* TO 'habitflow_demo'@'%';
FLUSH PRIVILEGES;
```

如果用同学电脑作为数据库主机，还需要确认：

- MySQL 监听地址允许局域网访问。
- Windows 防火墙放行 3306 端口。
- 大家连接的是同一个局域网。
- 不要直接共享 root 账号。

## 中期展示最优解

中期展示优先展示 API 和核心链路，不建议现场临时从零注册造数据。

推荐流程：

1. 演示前运行造数脚本，生成固定账号和业务数据。
2. 登录 `member3 / 123456`，展示成员三负责的社交内容。
3. 走一遍核心链路：登录、目标、打卡、统计、好友、圈子、帖子、点赞、评论。
4. 用 Swagger 或接口文档证明前后端是通过 API 联通的。

演示账号：

| 用户名 | 密码 | 用途 |
| --- | --- | --- |
| demo_admin | 123456 | 展示完整目标、统计、社交数据 |
| member1 | 123456 | 好友、圈子成员、评论互动 |
| member2 | 123456 | 好友申请、评论互动 |
| member3 | 123456 | 成员三演示账号，重点展示社交功能 |

## 一键填充演示数据

先确保 MySQL 可连接，并安装后端依赖：

```powershell
cd backend_fastapi
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

如果本机密码是 `123456`：

```powershell
$env:DATABASE_URL="mysql+pymysql://root:123456@localhost:3306/habitflow?charset=utf8mb4"
```

运行脚本：

```powershell
.\.venv\Scripts\python.exe .\scripts\seed_demo_data.py --reset-demo
```

`--reset-demo` 只会重置固定演示账号和相关演示数据，不会清空真实用户表。

## 成功标准

数据库协作成功标准：

- 三个人使用同一个 `DATABASE_URL` 后，能看到同一批用户、目标、圈子和帖子。
- A 用户发布帖子后，B 用户刷新圈子动态能看到。
- A 用户点赞或评论后，B 用户看到点赞数、评论数变化。
- 新建目标、打卡、统计看板数据能落库并再次查询。

中期展示成功标准：

- 前端页面能正常打开。
- `member3 / 123456` 能登录。
- 目标列表、打卡记录、统计看板有数据。
- 社交圈子能看到成员、帖子、点赞和评论。
- Swagger `/docs` 中可以直接调用核心 API。

npm install
