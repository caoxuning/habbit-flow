# HabitFlow 数据库设计文档

> 版本：v2.0（基于云数据库实际结构）
> 数据库：MySQL 8.0+
> 字符集：utf8mb4

---

## 数据表总览（共13张表）

| 序号 | 表名 | 中文名 | 功能说明 |
|:---:|:---|:---|:---|
| 1 | `user` | 用户表 | 存储账号与密码 |
| 2 | `goal` | 目标表 | 用户创建的打卡目标 |
| 3 | `check_in` | 打卡记录表 | 每次打卡的具体记录 |
| 4 | `badge` | 勋章定义表 | 系统所有勋章的配置 |
| 5 | `user_badge` | 用户勋章表 | 用户获得的勋章记录 |
| 6 | `notification` | 消息通知表 | 系统推送的通知消息 |
| 7 | `friendship` | 好友关系表 | 用户间的好友申请与关系 |
| 8 | `social_circle` | 打卡圈子表 | 圈子定义（如英语圈、健身圈） |
| 9 | `circle_member` | 圈子成员表 | 用户与圈子的关联关系 |
| 10 | `circle_post` | 圈子帖子表 | 圈子内的动态/帖子 |
| 11 | `post_like` | 帖子点赞表 | 帖子的点赞记录 |
| 12 | `post_comment` | 帖子评论表 | 帖子的评论记录 |
| 13 | `direct_message` | 私信表 | 用户间的私信 |

---

## 各表详细设计

### 1. user - 用户表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 用户编号 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码（加密存储） |
| email | VARCHAR(120) | DEFAULT NULL | 邮箱 |
| create_time | DATETIME | NOT NULL | 创建时间 |

### 2. goal - 目标表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 目标编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 用户编号 |
| name | VARCHAR(120) | NOT NULL | 目标名称 |
| type | VARCHAR(50) | NOT NULL | 目标类型（运动/英语/阅读/学习等） |
| start_date | DATE | NOT NULL | 开始日期 |
| end_date | DATE | NOT NULL | 结束日期 |
| cycle | VARCHAR(20) | NOT NULL | 目标周期：DAILY/WEEKLY/MONTHLY |
| daily_target_count | INT | NOT NULL, DEFAULT 1 | 每日目标次数 |
| **priority** | **VARCHAR(20)** | **NOT NULL, DEFAULT 'NORMAL'** | **优先级：LOW/NORMAL/HIGH/URGENT** |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'ACTIVE' | 状态：ACTIVE/PAUSED/DONE |
| create_time | DATETIME | NOT NULL | 创建时间 |
| update_time | DATETIME | NOT NULL | 更新时间 |

> **注意**：`priority` 字段是云数据库中存在但旧版 schema.sql 缺失的字段。

### 3. check_in - 打卡记录表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 打卡编号 |
| goal_id | BIGINT | FK → goal.id, NOT NULL | 目标编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 用户编号 |
| check_date | DATE | NOT NULL | 打卡日期 |
| check_time | DATETIME | NOT NULL | 打卡时间（精确到时分秒） |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'DONE' | 打卡状态 |
| remark | VARCHAR(500) | DEFAULT NULL | 备注内容 |
| makeup | TINYINT(1) | NOT NULL, DEFAULT 0 | 是否补卡：0=正常, 1=补卡 |

### 4. badge - 勋章定义表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 勋章编号 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 勋章编码（程序判断用） |
| name | VARCHAR(80) | NOT NULL | 勋章名称 |
| description | VARCHAR(255) | NOT NULL | 勋章描述 |
| condition_text | VARCHAR(255) | NOT NULL | 获取条件（前台展示） |

**种子数据：**

| code | 名称 | 条件 |
|:---|:---|:---|
| FIRST_CHECK_IN | 初次打卡勋章 | 累计打卡 1 次 |
| STREAK_7 | 连续7天勋章 | 连续打卡 7 天 |
| STREAK_30 | 连续30天勋章 | 连续打卡 30 天 |
| CHECK_IN_100 | 百次打卡勋章 | 累计打卡 100 次 |
| DISCIPLINE_MASTER | 自律达人勋章 | 累计100次且连续30天 |

### 5. user_badge - 用户勋章关联表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 记录编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 用户编号 |
| badge_id | BIGINT | FK → badge.id, NOT NULL | 勋章编号 |
| obtained_time | DATETIME | NOT NULL | 获得时间 |

### 6. notification - 消息通知表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 通知编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 接收用户编号 |
| type | VARCHAR(50) | NOT NULL | 通知类型（BADGE/FRIEND/SYSTEM） |
| title | VARCHAR(120) | NOT NULL | 通知标题 |
| content | VARCHAR(500) | NOT NULL | 通知内容 |
| is_read | TINYINT(1) | NOT NULL, DEFAULT 0 | 是否已读：0=未读, 1=已读 |
| create_time | DATETIME | NOT NULL | 创建时间 |

> **注意**：此表在云数据库中存在但旧版 schema.sql 缺失。

### 7. friendship - 好友关系表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 好友关系编号 |
| requester_id | BIGINT | FK → user.id, NOT NULL | 申请方用户编号 |
| addressee_id | BIGINT | FK → user.id, NOT NULL | 接收方用户编号 |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'PENDING' | 状态：PENDING/ACCEPTED/REJECTED |
| message | VARCHAR(255) | DEFAULT NULL | 申请留言 |
| create_time | DATETIME | NOT NULL | 创建时间 |
| update_time | DATETIME | NOT NULL | 更新时间 |

### 8. social_circle - 打卡圈子表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 圈子编号 |
| name | VARCHAR(80) | NOT NULL, UNIQUE | 圈子名称 |
| description | VARCHAR(255) | NOT NULL | 圈子简介 |
| icon | VARCHAR(20) | NOT NULL, DEFAULT 'TAG' | 圈子图标标识（EN/FIT/READ/TAG） |
| owner_user_id | BIGINT | FK → user.id, DEFAULT NULL | 圈主编号 |
| member_count | INT | NOT NULL, DEFAULT 0 | 成员数量（冗余字段） |
| create_time | DATETIME | NOT NULL | 创建时间 |

**种子数据：** 英语打卡圈(EN)、健身打卡圈(FIT)、阅读打卡圈(READ)

### 9. circle_member - 圈子成员表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 成员编号 |
| circle_id | BIGINT | FK → social_circle.id, NOT NULL | 圈子编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 用户编号 |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'MEMBER' | 角色：OWNER/MEMBER |
| join_time | DATETIME | NOT NULL | 加入时间 |

### 10. circle_post - 圈子帖子表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 帖子编号 |
| circle_id | BIGINT | FK → social_circle.id, NOT NULL | 所属圈子编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 发帖人编号 |
| **check_in_id** | **BIGINT** | **FK → check_in.id, DEFAULT NULL** | **关联打卡编号（打卡分享类型）** |
| **visibility** | **VARCHAR(20)** | **NOT NULL, DEFAULT 'PUBLIC'** | **可见性：PUBLIC/MEMBER_ONLY** |
| **post_type** | **VARCHAR(30)** | **NOT NULL, DEFAULT 'NORMAL'** | **帖子类型：NORMAL/CHECKIN_SHARE** |
| content | VARCHAR(1000) | NOT NULL | 帖子内容 |
| create_time | DATETIME | NOT NULL | 发布时间 |

> **注意**：`check_in_id`、`visibility`、`post_type` 是云数据库中存在但旧版 schema.sql 缺失的字段。

### 11. post_like - 帖子点赞表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 点赞编号 |
| post_id | BIGINT | FK → circle_post.id, NOT NULL | 帖子编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 点赞用户编号 |
| create_time | DATETIME | NOT NULL | 点赞时间 |

### 12. post_comment - 帖子评论表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 评论编号 |
| post_id | BIGINT | FK → circle_post.id, NOT NULL | 帖子编号 |
| user_id | BIGINT | FK → user.id, NOT NULL | 评论用户编号 |
| content | TEXT | NOT NULL | 评论内容 |
| create_time | DATETIME | NOT NULL | 评论时间 |

### 13. direct_message - 私信表

| 字段 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | BIGINT | PK, AUTO_INCREMENT | 私信编号 |
| sender_id | BIGINT | FK → user.id, NOT NULL | 发送方用户编号 |
| receiver_id | BIGINT | FK → user.id, NOT NULL | 接收方用户编号 |
| content | TEXT | NOT NULL | 私信内容 |
| create_time | DATETIME | NOT NULL | 发送时间 |

> **注意**：`direct_message` 表在云数据库中存在但旧版 schema.sql 缺失。

## 数据库关系图（ER描述）

```
user ──┬── goal (1:N)          用户拥有多个目标
       ├── check_in (1:N)      用户有多次打卡记录
       ├── user_badge (1:N)    用户获得多个勋章
       ├── notification (1:N)  用户收到多个通知
       ├── friendship (1:N)    用户作为申请方/接收方
       ├── circle_member (1:N) 用户加入多个圈子
       ├── circle_post (1:N)   用户发布多个帖子
       ├── post_like (1:N)     用户点赞多个帖子
       ├── post_comment (1:N)  用户评论多个帖子
       ├── direct_message as sender (1:N)
       └── direct_message as receiver (1:N)

goal ──┬── check_in (1:N)      每个目标有多次打卡
       └── circle_post (1:N)   目标打卡可分享为帖子

social_circle ──┬── circle_member (1:N)   圈子有多个成员
                └── circle_post (1:N)     圈子有多个帖子

circle_post ──┬── post_like (1:N)         帖子有多个点赞
              └── post_comment (1:N)      帖子有多个评论
```

| **circle_post** | ❌ 缺失 `check_in_id`、`visibility`、`post_type` 字段 |
| **—** | ❌ **完全缺失 `notification` 表** |
| **—** | ❌ **完全缺失 `direct_message` 表** |
