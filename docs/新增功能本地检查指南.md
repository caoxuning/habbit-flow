# HabitFlow 新增功能本地检查指南

本文档用于在本地电脑检查以下 3 个新增功能是否正常：

- 内容推荐系统：打卡成功后弹出与目标类型匹配的推荐内容
- 成长日志时间线：按日期倒序展示打卡、勋章、每日推荐，并支持筛选
- 数据导出：在打卡记录页面导出 Excel 文件

## 1. 启动前准备

### 1.1 检查 MySQL

确认本机 MySQL 已启动，并且存在 `habitflow` 数据库。

如果是第一次运行，可以在项目根目录执行数据库脚本：

```powershell
cd C:\Users\Lenovo\habbit-flow
mysql -u root -p < database\schema.sql
```

后端 FastAPI 默认连接地址在 `backend_fastapi/app/database.py`：

```text
mysql+pymysql://root:1234@localhost:3306/habitflow?charset=utf8mb4
```

如果你的 MySQL 密码不是 `1234`，推荐临时设置 `DATABASE_URL` 后再启动后端：

```powershell
cd C:\Users\Lenovo\habbit-flow\backend_fastapi
$env:DATABASE_URL="mysql+pymysql://root:你的密码@localhost:3306/habitflow?charset=utf8mb4"
```

### 1.2 安装后端依赖

```powershell
cd C:\Users\Lenovo\habbit-flow\backend_fastapi
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

本次数据导出功能需要 `openpyxl`。如果启动时报：

```text
ModuleNotFoundError: No module named 'openpyxl'
```

重新执行上面的安装命令即可。

### 1.3 检查前端依赖

```powershell
cd C:\Users\Lenovo\habbit-flow\frontend
npm.cmd install
```

如果 PowerShell 提示 `npm.ps1 cannot be loaded because running scripts is disabled`，使用 `npm.cmd`，不要直接用 `npm`。

## 2. 启动项目

### 2.1 启动后端

打开一个 PowerShell 窗口：

```powershell
cd C:\Users\Lenovo\habbit-flow\backend_fastapi
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8081
```

启动成功后访问：

```text
http://localhost:8081/docs
```

如果能看到 FastAPI 接口文档，说明后端已启动。

### 2.2 启动前端

再打开一个 PowerShell 窗口：

```powershell
cd C:\Users\Lenovo\habbit-flow\frontend
npm.cmd run dev
```

启动成功后访问：

```text
http://localhost:5173
```

前端 Vite 代理已配置为把 `/api` 转发到：

```text
http://localhost:8081
```

所以后端端口需要保持 `8081`，除非你同步修改 `frontend/vite.config.js`。

## 3. 基础功能冒烟检查

先确认旧功能正常，再检查新功能。

1. 打开 `http://localhost:5173`。
2. 注册一个测试账号，或使用已有账号登录。
3. 进入「目标管理」。
4. 点击「新建目标」。
5. 建议创建这些目标类型之一，方便触发匹配推荐：

```text
英语
运动
阅读
学习
健身
早睡
```

示例：

```text
目标名称：每天背 20 个单词
目标类型：英语
开始日期：今天
结束日期：下个月
目标周期：每日
每日目标次数：1
状态：进行中
```

保存后，如果目标列表能看到新目标，说明登录、目标创建、前后端联调基本正常。

## 4. 检查新增功能 ①：内容推荐系统

### 4.1 页面操作检查

1. 进入「打卡记录」或「目标管理」。
2. 对刚创建的目标执行「今日打卡」。
3. 打卡成功后，应自动弹出一个内容推荐弹窗。

预期效果：

- 弹窗标题类似「打卡成功 · 精选内容」。
- 英语目标会出现名人名言、英语词汇、句型、学习技巧等内容。
- 运动、阅读、学习、健身、早睡目标会出现对应类型的推荐内容。
- 如果有同类型目标的历史备注，弹窗中可能出现「同路人寄语」。

### 4.2 接口返回检查

可以在浏览器开发者工具的 Network 面板中查看：

```text
POST /api/check-ins
```

成功响应的 `data` 中应包含：

```json
{
  "checkIn": {},
  "inspiration": {
    "goalType": "英语",
    "content": "...",
    "type": "quote",
    "cn": "...",
    "example": "..."
  },
  "peerTips": []
}
```

### 4.3 常见情况

如果提示「该目标当天已打卡」，说明同一个目标今天已经打过卡。可以换一个目标测试，或等第二天再测。

如果没有弹窗但打卡成功，重点检查：

- 后端 `POST /api/check-ins` 返回里是否有 `inspiration`
- 前端控制台是否有 JavaScript 报错
- 当前是否调用的是「补卡」，补卡接口目前只返回打卡记录，不弹推荐内容

## 5. 检查新增功能 ②：成长日志时间线

### 5.1 左侧菜单入口

登录后看左侧菜单，应出现：

```text
成长日志
```

图标为 Timer 风格图标。

点击后进入成长日志页面。

### 5.2 时间线展示检查

完成至少一次打卡后，进入「成长日志」。

预期效果：

- 时间线按日期倒序展示，最新日期在上面。
- 打卡事件显示目标名称、备注、补卡标签。
- 获得勋章时显示勋章事件。
- 有打卡的日期会附带一条每日推荐内容。

### 5.3 筛选检查

页面右上角筛选项应包含：

```text
7天
30天
90天
全部
```

逐个切换后，页面应重新加载时间线。

对应接口：

```text
GET /api/stats/timeline?days=7
GET /api/stats/timeline?days=30
GET /api/stats/timeline?days=90
GET /api/stats/timeline?days=0
```

其中 `days=0` 表示全部记录。

### 5.4 数据概览页预览区块

进入「数据概览」页面，图表下方应看到：

```text
成长日志预览
```

预期效果：

- 最多展示前 7 条时间线日期记录。
- 点击「查看全部」后跳转到「成长日志」页面。

## 6. 检查新增功能 ③：数据导出

### 6.1 页面操作检查

1. 进入「打卡记录」页面。
2. 确认当前账号至少有 1 条打卡记录。
3. 点击顶部工具栏的「导出打卡记录」按钮。

预期效果：

- 浏览器开始下载 `.xlsx` 文件。
- 文件名类似：

```text
打卡记录_2026-06-18.xlsx
```

- 用 Excel 或 WPS 打开后，应看到这些列：

```text
日期
目标名称
目标类型
备注内容
打卡类型
打卡时间
```

### 6.2 接口检查

对应接口：

```text
GET /api/exports/checkins?format=xlsx
```

请求需要登录态，也就是需要带：

```text
Authorization: Bearer 你的token
```

响应类型应为：

```text
application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

### 6.3 常见情况

如果提示「没有符合条件的打卡记录」，说明当前登录账号还没有打卡数据。先完成一次今日打卡，再重新导出。

如果点击按钮无反应，检查浏览器是否拦截下载，或打开开发者工具查看 `/api/exports/checkins` 是否返回错误。

## 8. 常见启动问题排查

### 8.1 PowerShell 执行策略导致 npm 不能跑

报错示例：

```text
npm.ps1 cannot be loaded because running scripts is disabled
```

处理方式：

```powershell
npm.cmd run dev
npm.cmd run build
```

### 8.2 数据库连接失败

常见报错：

```text
Access denied for user 'root'@'localhost'
Unknown database 'habitflow'
Can't connect to MySQL server
```

处理方式：

- 确认 MySQL 服务已启动。
- 确认数据库 `habitflow` 已创建。
- 确认 `DATABASE_URL` 中用户名、密码、端口正确。

### 8.3 端口被占用

后端默认端口：

```text
8081
```

前端默认端口：

```text
5173
```

如果 `8081` 被占用，可以换端口启动后端，例如 `8090`：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8090
```

但同时需要修改 `frontend/vite.config.js` 中代理地址：

```js
target: 'http://localhost:8090'
```

### 8.4 终端出现中文乱码

如果只是 PowerShell 输出中文乱码，但页面显示正常、接口正常，这通常不是功能错误。

可以尝试在当前 PowerShell 窗口执行：

```powershell
chcp 65001
```

然后重新启动服务。
